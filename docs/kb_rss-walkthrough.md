# Walkthrough - `kb-rss` Implementation & UAT Curation Explorer

I have implemented the complete `kb-rss` package under `remotes/kb-rss` using a clean, modular structure. This package provides an AI-curated RSS feeds system with a Typer CLI, a background service, an Electron desktop client (retro cream theme, pinned sidebar), and a daily Ollama `gemma4` curation agent.

---

## 1. Python Watcher, CLI & Ollama Agent

- **Data Models (`models.py`)**: Defined Pydantic schema wrappers for `RssSource`, `RssCategory`, `RssFeed`, and `FeedItemEntry`.
- **Database Schema (`db.py`)**: Built migration and management helpers using `sqlite-utils` for tables:
  - `rss_feeds` (stores feed metadata, XML URLs, and descriptions).
  - `rss_feed_entries` (stores parsed posts and user interactions like `liked`, `favorite`, `comment`, `clicked`, and `shared`).
  - `rss_categories` (stores user feed categories).
  - `rss_feed_categories` (many-to-many junction mapping).
  - `rss_daily_reports` (stores daily curation digests generated by Ollama).
- **RSS Watcher (`watcher.py`)**: Implemented the feed parser engine using `feedparser`. Polling loop updates feed entries every 5 minutes and handles per-feed exceptions gracefully.
- **Ollama Taste Agent (`agent.py`)**:
  - **Taste Updater**: Explores user interaction history (likes, dislikes, comments, click statistics) and updates/saves taste preferences to `~/.kb/agent_user_tastes.md`.
  - **Daily Curation Digest**: Curates the 50 most recent unread entries based on the profile, asks Ollama to select the top 5-10 with a 1-sentence rationale, generates a markdown report, saves the entries, and pushes a Gotify notification using `kb_core.notifier`.
- **CLI Commands (`cli.py`)**: Exposes subcommands:
  - `watch`: foreground feed ingestion loop.
  - `poll-once`: one-off feeds polling update.
  - `start`/`stop`/`status`: background daemon management (running silently via `pythonw.exe` on Windows).
  - `install`: installs startup logon batch script `start_kb_rss.cmd` to Windows Startup folder (downloads prebuilt desktop binary from GitHub Releases if not found).
  - `feed add`/`remove`/`list`: dynamic CLI feeds administration.
  - `category add`/`list`: category metadata utilities.
  - `import-json`: seeds feeds database from JSON configurations (like `rss_feeds.json`).
  - `agent-run`: manually triggers taste updates and curation digest generation.
  - `serve`: spawns Electron app (downloads prebuilt desktop binary from GitHub Releases if not found).
  - `update`: queries GitHub release channel and downloads the latest prebuilt Electron executable.
- **Unit Tests (`tests/`)**: Built tests with 100% pass rate covering db operations, mock feedparser ingestion, and mock Ollama HTTP payload responses.

---

## 2. Electron Tray & Pinned React/Tailwind Client (`desktop/`)

- **Direct SQLite Bindings**: Electron `main.js` communicates directly with SQLite `~/.kb/kb.db` for instant queries.
- **Tray Menu**: Shows the 5 most recent articles. Clicking opens the article directly in the user's default web browser and increments the database `clicked` counter.
- **Client UI (`App.jsx`)**: Styled in an elegant retro-solarized cream design with high accessibility:
  - **Pinned Layout**: left sidebar remains fixed in place when scrolling.
  - **Curation Feed**: paginated feed of articles with instant likes, stars, comment editor, and shares/clicks tracking.
  - **Daily AI Digest**: sidebar to select report dates. Loads AI summary curations and recommended articles with AI curatorial explanations.
  - **Feeds Manager**: form to register feed endpoints dynamically and edit categories.
  - **Taste Editor**: inline raw text markdown editor for editing `user_interests.md` and viewing the generated `agent_user_tastes.md` profile.
  - **Python Bridge**: IPC handles executing python CLI actions directly (e.g. running agent, polling feeds, deleting/adding sources).

## UAT (User Acceptance Testing) Instructions

Please perform User Acceptance Testing (UAT) in your environment using the following steps:

### Step 1: Initialize Database & Seed Feeds
Open a terminal inside `remotes/kb-rss` and run:
```bash
uv run kb-rss import-json rss_feeds.json
```
Verify that the database tables are initialized, feeds are fetched, and categories (general, news, tech_news, coding) are seeded.

### Step 2: Poll for Feed Updates
Update the articles from the command line:
```bash
uv run kb-rss poll-once
```
Verify it prints the count of new entries added from your feed endpoints.

### Step 3: Run background watcher daemon
Launch the silent daemon:
```bash
uv run kb-rss start
```
Check if it's active:
```bash
uv run kb-rss status
```
Then stop it:
```bash
uv run kb-rss stop
```

### Step 4: Launch Electron App in Production Mode
Ensure you compile the latest build before launching the Electron app to see the correct compiled code:
```bash
# Build Vite production assets
cd desktop
npm run build
cd ..

# Launch production app
uv run kb-rss serve
```

Inside the UI, explore and test:
1. **Interactive Feed**: Like some articles, star/favorite others, and write comments.
2. **Dynamic Curation**: Edit `user_interests.md` in the "Taste Profile" tab and save it.
3. **Daily Digest Curation**: Click the **Generate AI Digest** button at the top right.
   - If Ollama is offline or misconfigured, it will now return a clean error toast notifying you of the failure (rather than failing silently).
   - If successful, it curates recommendations in the "Daily AI Digest" tab and triggers Gotify notifications.
4. **Embedded Webview & Raw HTML Export**: Select an article and open the **Web Preview** tab in the drawer. Once the page loads, switch back to the **Summary Mode** tab. It will now indicate "Raw HTML Offline Copy Available" and render the exported page safely inside an isolated sandboxed iframe! You can verify that the raw HTML has been stored in the `full_content` database column.
5. **Offline Reader View**: For articles where full text has not yet been fetched, click **Load Reader View** to trigger the Python parser, clean the document, extract a preview thumbnail, and save it.
6. **Daemon Status Verification**: Run `uv run kb-rss status` and `uv run kb-clipboard status` in the terminal to verify that the CLI background watcher status commands now correctly check Windows PIDs (using ctypes) and report accurate states.
7. **Feed Manager**: Try adding a new feed (e.g. `https://news.ycombinator.com/rss` under category `tech_news`) and verify it polls and inserts new articles.
